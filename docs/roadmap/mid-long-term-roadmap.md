# Mid/Long-Term Roadmap

Last updated: 2026-02-15
Owner: Kakio / Eduitit

## 목적
- 단기 핫픽스 중심 운영에서 벗어나, 성능/안정성/개발 생산성을 중장기적으로 개선한다.

## 우선순위 원칙
1. 사용자 영향이 큰 안정성 이슈 우선
2. 운영 비용을 줄이는 자동화 우선
3. 큰 변경은 무중단 전환 + 롤백 경로를 먼저 설계

## 트랙 A: 운영 안정성
### A1. Cron/배치 내구성 고도화
- 상태: 진행중
- 목표: 실패 격리, 재시도 전략, 실패 알림 표준화
- 산출물:
  - 배치 명령 공통 에러 처리 가이드
  - 실패 알림(로그/웹훅) 규칙
  - 운영 Runbook

### A2. 구조화 로깅 확장
- 상태: 진행중
- 목표: Request ID 기반 추적 일원화
- 산출물:
  - 앱 전역 로그 포맷 통일
  - 주요 비즈니스 이벤트 로그 키 표준

## 트랙 B: 프론트엔드/성능
### B1. Tailwind 빌드 전환 (기존 항목 #3)
- 상태: 보류 (대규모 변경)
- 목표: CDN 의존 축소, 빌드 기반 CSS 최적화
- 단계:
  1. 현황 조사(CDN 사용처, 동적 클래스 목록)
  2. 빌드 체인 병행 도입
  3. safelist 설계 및 검증
  4. 점진 전환(동시 로딩 -> CDN 제거)
- 리스크:
  - 동적 클래스 purge 누락으로 UI 깨짐
- 롤백:
  - base 템플릿에서 CDN 즉시 재활성

### B2. 성능 예산 수립
- 상태: 계획
- 목표: 모바일 체감 성능 개선
- KPI 예시:
  - LCP/INP/CLS 기준선 설정
  - 이미지 lazy loading 커버리지 90%+

## 트랙 C: 품질 게이트
### C1. A11y 자동 점검 고도화
- 상태: 진행중
- 목표: critical 0 유지 + 경고 지속 감소
- 산출물:
  - check_a11y 개선(오탐 감축)
  - PR 체크리스트 반영

### C2. 배포 전 검증 파이프라인
- 상태: 계획
- 목표: 운영 반영 전 기본 체크 자동화
- 산출물:
  - manage.py check
  - check_a11y
  - 핵심 스모크 테스트

## 분기 운영 리듬 (권장)
- 매주: 운영 이슈 triage
- 격주: 보류 과제 재평가
- 매월: KPI 리포트 + 다음 달 우선순위 재조정

2. check_a11y 경고 상위 20개 템플릿 정리
3. 배포 전 검증 명령을 스크립트로 통합
