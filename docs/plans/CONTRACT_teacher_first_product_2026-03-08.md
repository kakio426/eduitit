# Teacher-First Product Contract

작성일: 2026-03-08
적용 범위: Eduitit 전사 교사형 서비스, 홈/카탈로그/가이드/워크플로우 통합

## 목적
- 교사는 서비스 이름이 아니라 `하려는 일`로 들어간다.
- 첫 화면은 `지금 하는 일 / 핵심 결과 / 보조 설정`만 보여준다.
- 공유·협업·연동·관리·운영은 기본적으로 접힘 또는 2차 위치다.
- 설명이 많이 필요한 구조는 실패다.
- 완료 기준은 `작동한다`가 아니라 `처음 쓰는 교사가 설명 없이 쓴다`이다.

## 전사 계약
1. 홈은 `오늘 바로 시작 / 내 작업대 / 새로 써보기 / 핵심 업무 / 더 많은 도구` 구조를 따른다.
2. 서비스 첫 화면의 주인공은 설명이 아니라 작업 영역이다.
3. 전면 primary action은 1~3개까지만 둔다.
4. 희소 기능과 관리 기능은 기본 화면에서 주 기능보다 앞에 나오면 안 된다.
5. 같은 목표 CTA를 여러 곳에서 중복 노출하지 않는다.
6. iframe 전체 페이지 임베드, 중첩 스크롤, 중복 헤더 같은 과도기 구조는 최종 teacher-facing UX로 금지한다.
7. 모바일은 읽기 전용이 아니라 `안전한 공통 작업만 허용` 원칙을 따른다.
8. 가이드/매뉴얼/외부 도구 소개 화면은 `1차 시작점`이 아니라 `보조 참고 화면`이다.

## 화면 역할 계약
- `홈`: 지금 할 일과 개인 작업대에서 바로 시작한다.
- `전체 서비스`: 업무 기준으로 전체를 탐색한다.
- `빠른 사용 안내`: 막힐 때만 짧게 확인한다.
- `외부 AI 도구 참고`: Eduitit 밖의 참고 도구를 비교한다.

## 기준선 상태
- 홈/카탈로그 teacher-first 기준선: 진행 중인 baseline 구현 존재
- 대표 서비스 첫 화면 표준화: `noticegen`, `timetable`, `parentcomm`, `reservations` 적용 baseline 존재
- `classroom_workspace + calendar + message`: 단일 달력 원칙으로 정리된 baseline 존재

## 완료 판단
- 교사가 홈에서 1번 클릭으로 핵심 서비스에 들어간다.
- 서비스 첫 화면에서 설명을 읽지 않아도 주요 작업이 가능하다.
- 가이드/도움말/관리 기능은 기본 화면을 잡아먹지 않는다.
- 테스트, 접근성, 스모크, 롤백 기준이 존재한다.
