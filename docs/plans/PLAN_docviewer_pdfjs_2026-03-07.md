# 문서 미리보기실(PDF.js) V1 계획

기준 문서:
- `claude.md`
- `SERVICE_INTEGRATION_STANDARD.md`

범위 선언:
- UI 범위: `app-level`
- target_app: `docviewer`
- do_not_touch_apps: `core`, `products`, `config`, 기존 서비스 앱 전체 (연결용 최소 등록 제외)

## 1. 한 줄 목표

교사는 PDF 하나만 올리면, 같은 화면에서 바로 넘겨 보고 확대하고 인쇄용 새 탭까지 이어서 처리할 수 있다.

## 2. 교사 흐름 와이어프레임

### A. 메인 화면
```text
+--------------------------------------------------------------+
| [문서 미리보기실]                                            |
| PDF를 올리면 같은 화면에서 바로 넘겨 보고 확인할 수 있어요. |
+-----------------------------+--------------------------------+
| [교사 작업 흐름]            | [미리보기 화면]               |
| 1. PDF 선택                 | 아직 열린 PDF가 없습니다.     |
| 2. 빠르게 검수              | PDF를 열면 여기서 바로 보임   |
| 3. 다음 행동 선택           |                                |
+-----------------------------+--------------------------------+
| [PDF 파일 선택]             | [이전 쪽] [1 / 4쪽] [다음 쪽] |
| [다른 PDF 고르기]           | [축소] [100%] [확대]          |
| [인쇄하기 (새 탭)]          | [PDF 첫 페이지 캔버스]        |
+-----------------------------+--------------------------------+
```

### B. 핵심 UX 계약
- PDF 선택 직후: 오른쪽 미리보기 즉시 갱신
- 렌더링 중: 로딩 상태 문구 표시
- 렌더링 후: 현재 쪽 / 확대 비율 / 다음 행동이 동시에 보임
- 오류 시: 상태 박스에서 바로 피드백

## 3. V1 포함 범위
- PDF 업로드(브라우저 메모리 기반)
- 첫 페이지 자동 렌더링
- 이전/다음 페이지 이동
- 확대/축소
- 인쇄용 새 탭 열기

## 4. V1 제외 범위
- 서버 저장
- 주석/메모
- 공유 링크
- 권한 협업
- 원격 PDF 프록시

## 5. 검증 기준
- `python manage.py check`
- `python manage.py test docviewer products.tests.test_ensure_docviewer`
- `node --check docviewer/static/docviewer/docviewer.js`
- 수동 스모크:
  1. 로그인 후 `/docviewer/` 진입
  2. PDF 선택
  3. 이전/다음 페이지 클릭
  4. 확대/축소 클릭
  5. 인쇄용 새 탭 버튼 클릭
