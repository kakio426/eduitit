# Teacher-First Mobile Policy

작성일: 2026-03-09

## Goal
- 모바일에서도 `읽기만`이 아니라 교사가 안전하게 끝낼 수 있는 공통 작업은 유지한다.
- 막힌 기능은 숨기지 말고 비활성화 + 짧은 이유 표시로 통일한다.

## Common Rules
- 허용: 단건 입력, 확인, 간단 수정, 다음 화면 이동, 단일 후속 액션
- 비허용: 대량 붙여넣기, 복잡한 구조 변경, 대량 대상 재정렬, 관리자 전용 고급 작업
- 막힌 기능은 무반응으로 두지 않는다. 비활성화 상태와 짧은 이유를 같이 둔다.
- 모바일에서는 드래그보다 탭/버튼 기반 조작을 우선한다.

## Safe / Unsafe Matrix
| 화면 | 모바일에서 허용 | 모바일에서 제한 |
| --- | --- | --- |
| 홈 / 내 작업대 | 도구 열기, 즐겨찾기 추가, 저장한 조합 적용, 정리 모드의 좌우 이동 버튼 | 드래그 정렬 |
| 교무수첩 / 달력 | 단건 입력, 일정 1건 추가, 메시지 붙여넣기 검토 | 대량 붙여넣기, 복잡한 표 구조 편집 |
| 안내문 / 학부모 소통 | 초안 확인, 후속 화면 이동, 단건 수정 | 관리자성 대량 발송 제어 |
| 동의 / 서명 | 초안 생성, 대상 확인, 상태 확인 | 대규모 대상 재구성 |
| 예약 / 일정 운영 | 결과 확인, 안내문/연락으로 후속 이동 | 관리자용 운영 설정 |

## Home / Workbench Rule
- `내 작업대`는 모바일에서도 같은 위치에 둔다.
- 순서 변경은 `작업대 정리` 후 카드 내부 좌우 이동 버튼으로만 처리한다.
- 키보드가 있는 환경에서는 좌우 화살표도 동일하게 동작한다.

## Exit Criteria
- 무반응 버튼 0
- 막힌 기능은 모두 이유 표시
- 모바일에서 메인 CTA와 후속 CTA 구분이 유지됨
