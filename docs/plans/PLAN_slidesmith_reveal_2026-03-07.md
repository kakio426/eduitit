# Slidesmith Plan

작성일: 2026-03-07
대상 앱: `slidesmith`
범위: `app-level`

## 목표
- 기존 교사용 서비스 톤을 유지한 발표 작성 화면을 만든다.
- 입력 -> 미리보기 -> 새 탭 발표 시작의 3단계 여정을 고정한다.
- reveal.js는 앱 정적 자산으로 고정해 외부 CDN 의존을 없앤다.

## 와이어프레임 체크
```text
+---------------------------------------------------------------+
| 상단: 뒤로가기 / 서비스 배지                                   |
+-----------------------------+---------------------------------+
| 좌측 작업 영역              | 우측 결과 영역                  |
| - 교사 발표 흐름 3단계      | - 슬라이드 수 / 예상 시간       |
| - 제목 입력                 | - 첫 슬라이드 / 다음 행동       |
| - 본문 입력(--- 구분선)     | - 표지 + 본문 슬라이드 카드     |
| - 템플릿 3종                | - 즉시 확인 가능한 순서         |
| - 미리보기 새로고침         |                                 |
| - 발표 시작 (새 탭)         |                                 |
+-----------------------------+---------------------------------+
```

## 구현 원칙
- 공용 레이아웃/공용 CSS는 수정하지 않는다.
- non-JS 환경에서도 `미리보기 새로고침`, `발표 시작 (새 탭)`이 동작하도록 POST 기반 폴백을 둔다.
- reveal.js 발표 화면은 별도 템플릿으로 분리해 기존 네비게이션과 충돌하지 않게 한다.

## 검증 계획
- `python manage.py check`
- `python manage.py test slidesmith products.tests.test_ensure_slidesmith`
- `node --check slidesmith/static/slidesmith/slidesmith.js`
- `node --check slidesmith/static/slidesmith/slidesmith_present.js`
- 브라우저 스모크: 편집 화면 로드 -> 템플릿 적용 -> 새 탭 발표 -> 슬라이드 이동 -> PDF 저장 안내 토글
