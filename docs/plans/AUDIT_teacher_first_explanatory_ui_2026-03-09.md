# Teacher-First Explanatory UI Audit

작성일: 2026-03-09

## Audit Rule
- 설명을 읽어야 이해되는 구조는 실패로 본다.
- 희소 기능이 주 기능보다 크면 실패로 본다.
- 관리/설정/공유가 첫 화면 주인공이면 실패로 본다.

| --- | --- | --- |

## Immediate Follow-up
- 홈과 교사형 서비스 3개에서 동일한 버튼 hierarchy를 먼저 맞춘다.
- Empty / Success / Error 상태를 한 줄 행동 중심으로 다시 확인한다.
- 설명 카드가 다시 커지면 문구를 덜어내기 전에 레이아웃부터 줄인다.

## Phase G Sample Rollout
- noticegen: 상단 단계를 큰 설명 칩이 아니라 작은 진행 pill로 축소하고, 결과/후속 작업 문구를 한 줄 행동 중심으로 줄였다.
- reservations: 날짜 이동과 관리 메뉴는 유지하되 헤더 카드를 더 얇게 만들고 안내 문구를 짧게 줄였다.
- parentcomm: 상단 수치 카드를 상태 pill로 줄이고, `오늘 할 일 / 쪽지 / 상담`만 전면에 두고 `알림장 / 연락처`는 보조 메뉴로 내려 첫 화면 밀도를 낮췄다.
- product_list / mini_card: 서비스 발견 화면도 작업명 우선 카드 문법으로 맞추고, 카탈로그 헤더 설명을 짧게 줄였다.
- service_guide_list: 빠른 사용 안내 카드에서 서비스명은 보조, 안내 제목과 한 줄 지원 문구는 전면으로 올렸다.
- hwpxchat / textbooks: 도구 설명과 영어 서비스 라벨을 줄이고, 업로드·저장·바로 쓰기 같은 교사 행동 문구를 전면으로 올렸다.
- consent/create_step1: 정책 안내는 compact 기준으로 줄이고, 수신자 후보와 자동 적용 설정은 기본 펼침 대신 필요한 때만 열게 정리했다.
- slidesmith: 상단 도움 pill과 다음 행동 카드처럼 반복되는 설명은 줄이고, 본문 입력과 미리보기 흐름이 먼저 보이게 정리했다.
- happy_seed/dashboard: 사용 안내는 주 CTA 옆 큰 버튼이 아니라 보조 링크로 낮추고, 교실 열기와 새 교실 만들기가 먼저 보이게 유지했다.
- happy_seed/classroom_detail: 교사용 설명서는 헤더 주 버튼에서 빼고 보조 링크로 낮춰, 교실 운영 액션이 먼저 보이게 정리했다.
- signatures/create: 참석자 후보는 기본 펼침 대신 details로 접고, 원본 복귀만 보조 CTA로 유지해 생성 화면이 먼저 보이게 정리했다.
- signatures/maker: 서명 제작은 서명 목록으로 같은 작업 복귀 라벨로 줄이고, 보관함 안내는 선택 사항 details 안으로 낮춰 이미지 저장이 먼저 보이게 정리했다.

## Home Top Rollback Rules
- home SNS는 `요약 카드`가 아니라 `피드형 미리보기`여야 한다. 글 제목/본문, 작성자, 시간, 썸네일 중 핵심 정보가 바로 보여야 한다.
- home SNS에서 `공지` 빨간 라벨은 실제 `notice` 게시글에만 쓴다. `news_link`는 `링크` 또는 중립 표현으로 다룬다.
- home SNS는 가짜 버튼을 두지 않는다. 카드 전체나 CTA는 실제 링크여야 하고, 아래에 별도 `SNS 더 보기` 블록을 다시 만들지 않는다.
- 내 작업대는 `도구명 우선` 구조를 유지한다. 긴 작업 문장은 제목이 아니라 보조문 1줄로만 노출한다.
- 내 작업대는 4열을 유지하더라도 카드 높이와 패딩을 조절해 빈 공간이 과하게 남지 않게 한다.
- 작업대 정리 방향 문구는 디바이스 배치와 일치해야 한다. 모바일은 `위/아래`, 데스크톱은 `왼쪽/오른쪽`이다.

## Remaining Exceptions
- 현재 즉시 수정이 필요한 teacher-first 예외 화면은 없음. 브라우저 smoke에서 재확인한다.
