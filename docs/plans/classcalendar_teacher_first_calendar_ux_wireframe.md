# ClassCalendar Teacher-First UX Wireframe

## 목표

- 교사가 설명 없이도 바로 오늘 일정과 필요한 날짜를 찾을 수 있는 캘린더
- PC와 모바일 모두에서 객체가 삐져나오지 않고, 한글 줄바꿈이 덜 어색한 화면
- 월간 화면은 요약, 날짜 상세는 별도 패널이라는 역할 분리

## 현재 문제 목록

1. 월간 셀 안에 정보가 과하게 들어가면 화면 폭이 좁을 때 줄바꿈과 잘림이 거칠어진다.
2. 작은 배지와 과한 tracking, `truncate` 위주의 타이포그래피가 한글 제목을 읽기 어렵게 만든다.
3. 오늘 화면은 여러 카드가 같은 무게로 쌓여서 "지금 뭐 하면 되는지"가 한눈에 들어오지 않는다.
4. 월간 그리드와 선택 날짜 상세의 역할이 섞이면 사용자가 어디서 확인하고 어디서 행동해야 하는지 헷갈린다.
5. 보조 기능이 주 화면과 비슷한 무게로 보이면 교사 입장에서는 캘린더보다 도구 모음처럼 느껴진다.

## 화면 통합안

- 월간 캘린더
  - 역할: 날짜 분포와 전체 흐름을 요약해서 보여준다.
  - 원칙: 날짜 숫자, 개수, 상태 점만 보여주고 상세 텍스트는 넣지 않는다.

- 날짜 상세 패널
  - 역할: 선택한 날짜의 일정, 메모, 할 일을 읽고 여는 곳이다.
  - 원칙: 상세 정보와 액션은 이 패널에 집중한다.

- 오늘 화면
  - 역할: 오늘 바로 처리할 항목을 빠르게 확인한다.
  - 원칙: 필터는 남기되 상단에서 한 덩어리로 정리하고, 카드 텍스트는 줄임보다 읽힘을 우선한다.

- 보조 도구
  - 역할: 메시지 보관함, 공유, 연동 같은 기능은 필요할 때 이동해서 쓴다.
  - 원칙: 첫 화면의 주인공이 되지 않는다.

## PC 와이어프레임

```text
[상단]
월간 캘린더
2026년 3월
오늘 3건 · 이번 달 18건

[우측 액션]
메시지 보관함 | 이전 | 다음 | 오늘 | 새 일정

[본문]
좌측: 월간 그리드
- 날짜 숫자
- 해당 날짜 총 항목 수
- 일정/메모/할 일 상태 점

우측: 선택 날짜 패널
- 선택 날짜
- 이날의 요약 문장
- 연결된 항목 리스트
  - 서비스 라벨
  - 상태
  - 제목
  - 일정/마감 정보
```

## 모바일 와이어프레임

```text
[상단]
2026년 3월
이전 | 다음 | 오늘 | 새 일정

[본문]
월간 그리드
- 날짜 숫자
- 항목 수
- 상태 점

[아래]
선택 날짜 카드
- 날짜
- 요약 문장
- 연결된 항목 리스트
```

## 시각/타이포 규칙

1. 본문용 폰트 느낌은 하나로 유지한다.
2. 작은 보조 라벨도 최소 `text-xs` 크기를 기본으로 한다.
3. 한글 제목은 `truncate`보다 2줄 이내 줄바꿈과 `word-break: keep-all`을 우선한다.
4. 캘린더 셀에서는 제목 본문을 숨기고 숫자와 상태만 남긴다.
5. 강조는 검정/슬레이트 기반으로 두고, 메모만 앰버 계열로 분리한다.

## 구현 체크리스트

- [ ] 월간 헤더에 월 요약 문장 노출
- [ ] 오늘 화면 상단 액션을 묶어서 정리
- [ ] 월간 셀에 요약 정보만 남기고 상세 텍스트 제거
- [ ] 선택 날짜 패널 제목/설명 텍스트의 잘림 완화
- [ ] 작은 배지 크기와 tracking 축소
- [ ] 한글 줄바꿈이 필요한 제목과 파일명에 `keep-all`/2줄 clamp 적용
- [ ] 상세 모달의 첨부명과 제목 잘림 완화
- [ ] PC/모바일에서 셀 높이와 패널 읽기 흐름 재점검
