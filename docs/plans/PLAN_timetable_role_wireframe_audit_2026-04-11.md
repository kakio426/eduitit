# TimeTable Role Wireframe Audit 2026-04-11

## 현재 문제 목록

1. `/timetable/` 메인 허브에서 `업무 담당 교사`의 실제 주 행동인 `회의 보드 열기`가 전면에 없다.
2. 메인의 학년 카드가 전부 같은 편집 화면으로만 들어가서 `편집`, `회의`, `공유` 분기가 숨겨진다.
3. `일반 교사`와 `업무 담당` 역할 문구가 섞여 있어 누가 어디로 들어가야 하는지 한 번에 읽히지 않는다.
4. 담임용 반 입력 링크 화면은 기능은 되지만, `주간 입력 -> 날짜별 일정 -> 입력 완료` 순서가 구조로 드러나지 않는다.
5. 확정본 보기와 교사용 모아보기는 생성되지만, 관리자 허브에서 미리 확인하거나 전달 흐름으로 이어지는 연결이 약하다.

## 남길 화면 / 내릴 화면 / 통합할 화면

### 남길 화면

- `/timetable/` 관리자 허브
- `/timetable/workspaces/<id>/` 관리자 편집기
- `/timetable/workspaces/<id>/meeting/` 업무 담당 교사 회의 보드
- `/timetable/edit/<token>/` 담임용 반 입력 화면
- `/timetable/share/<token>/` 일반 교사용 읽기 전용 시간표
- `/timetable/share/portal/<token>/` 교사용 모아보기

### 내릴 화면

- 메인 허브에서 `업무 담당 / 담임`처럼 두 역할을 한 카드에 섞는 표현
- 학년 카드 안에서 목적 없이 한 화면으로만 들어가게 만드는 단일 링크 구조

### 통합할 화면

- 메인 허브의 역할 진입을 `관리 교사`, `담임 교사`, `업무 담당 교사`, `일반 교사`로 다시 정리한다.
- 메인 학년 카드는 `편집`, `회의`, `공유`로 바로 갈 수 있게 연결을 드러낸다.

## 텍스트 와이어프레임

```text
[메인 허브]
관리 교사 시작
[새 학기 만들기] [진행 중 시간표 보기]

학년 카드
- 편집 열기
- 회의 보드
- 확정본 보기(있을 때)

역할별 바로 열기
- 담임 교사: 받은 반 입력 링크 열기
- 업무 담당 교사: 학년 선택 -> 회의 보드 열기
- 일반 교사: 받은 확정본 링크 열기 / 확정본 미리보기
```

```text
[담임용 반 입력 링크]
반 이름 / 상태
입력 순서
1 주간 기본 시간표
2 날짜별 일정
3 입력 완료

[임시 저장] [입력 완료]
```

## 구현 체크리스트

- 메인 허브에 `업무 담당 교사` 전용 회의 진입 카드를 추가한다.
- 메인 학년 카드에 `편집`, `회의`, `확정본 보기` 빠른 진입을 드러낸다.
- 역할 문구를 `담임`, `업무 담당`, `일반 교사` 기준으로 분리한다.
- 담임용 반 입력 화면 상단에 입력 순서를 칩 형태로 드러낸다.
- 관리자 허브에서 published workspace의 읽기 전용 미리보기를 바로 열 수 있게 한다.
- 역할별 시작 흐름에 대한 뷰 테스트를 추가한다.
