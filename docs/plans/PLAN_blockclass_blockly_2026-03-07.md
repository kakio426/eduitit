# Blockclass Plan

작성일: 2026-03-07
대상 앱: `blockclass`
범위: `app-level`

## 목표
- Blockly를 로컬 정적으로 고정한 교사용 블록 실습 화면을 만든다.
- 템플릿 선택 -> 블록 배치 -> JSON/이미지 저장의 3단계 여정을 고정한다.
- 서버 저장 없이 브라우저 단독으로 완료되는 범위만 포함한다.

## 와이어프레임 체크
```text
+---------------------------------------------------------------+
| 상단: 뒤로가기 / 서비스 배지                                   |
+-----------------------------+---------------------------------+
| 좌측 조작 영역              | 우측 작업 영역                  |
| - 교사 활동 흐름 3단계      | - Blockly 워크스페이스          |
| - 순서/조건/반복 템플릿     | - 블록 수 / 상단 흐름 수        |
| - JSON 저장                 | - 상태 / 코드 보기 / 다음 행동  |
| - JSON 불러오기             | - 코드 미리보기                 |
| - 활동판 저장 (.svg)        |                                 |
+-----------------------------+---------------------------------+
```

## 구현 원칙
- 공용 레이아웃/공용 CSS는 수정하지 않는다.
- Blockly 자산은 앱 정적 경로에 복사해 외부 CDN 의존을 없앤다.
- 로컬 JSON 저장/불러오기와 SVG 저장만 제공하고, 서버 저장/제출은 넣지 않는다.

## 검증 계획
- `python manage.py check`
- `python manage.py test blockclass products.tests.test_ensure_blockclass`
- `node --check blockclass/static/blockclass/blockclass.js`
- 브라우저 스모크: 화면 로드 -> 템플릿 변경 -> JSON 저장 -> SVG 저장 -> JSON 불러오기 -> 코드 갱신
