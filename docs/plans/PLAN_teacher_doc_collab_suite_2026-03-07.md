# 교사용 공동 문서 허브 구현 계획 (2026-03-07)

기준 문서:
- `claude.md`
- `SERVICE_INTEGRATION_STANDARD.md`

범위 선언:
- UI 범위: `global`
- 대상 서비스 방향: 신규 독립 앱 + 기존 문서 자산 재사용

제안 서비스명:
- 교사용 노출명: `함께문서실`
- 카테고리: `문서·작성`
- 역할: 한글/한셀, Google Docs/Sheets, 문서 버전관리, 교사 협업을 하나의 허브에서 다루는 문서 중심 서비스

---

## 1. 한 줄 목표

교사는 문서 종류만 고르면, `eduitit`가 가장 알맞은 편집 방식을 바로 연결하고, 저장/공유/버전/권한/최신본 관리를 대신 처리한다.

---

## 2. 왜 이 방향이 맞는가

교사 문서 작업은 실제로 두 갈래다.

1. 공식 양식, 공문, 교육청 제출 문서
- 여전히 `한글/한셀`이 중심이다.
- 교사 PC에 이미 한컴이 설치되어 있으므로, 브라우저 안에서 새 편집기를 강제하는 것보다 `내 컴퓨터의 한글/한셀로 바로 열기`가 자연스럽다.

2. 명단, 분담표, 회의 메모, 공동 체크리스트, 학년 공유 목록
- 여기서는 `Google Docs/Sheets`의 실시간 공동편집이 훨씬 편하다.
- 특히 여러 교사가 동시에 수정하는 목록형 업무는 Google Sheets가 가장 빠르게 가치를 낸다.

따라서 `한컴과 Google을 한 서비스 안에서 동시에 제공`하는 것은 매우 좋은 방향이다. 다만 화면에서 둘을 섞어 혼란을 만들면 안 된다.

핵심 원칙:
- `공식 문서`는 한글/한셀 중심
- `실시간 목록 협업`은 Google Docs/Sheets 중심
- 둘 다 `eduitit` 안에서 만들고, 찾고, 권한을 나누고, 최신본을 관리하게 만든다

---

## 3. eduitit 안에서 재사용할 기존 자산

현재 저장소 기준으로 바로 활용 가능한 기반은 이미 있다.

1. 제품/매뉴얼 노출 기반
- `products` 앱의 `Product`, `ServiceManual`, `ManualSection`
- 신규 서비스도 같은 방식으로 카드, 미리보기, 이용방법까지 동시에 노출 가능

2. 교사 협업 권한 패턴
- `classcalendar`의 협업자 모델과 공유 설정 패턴
- "읽기", "편집", "학년 공유", "학급 공유" 같은 권한 구조에 재사용 가능

3. 문서 버전관리 기반
- `version_manager`의 문서 패키지, 버전, 공유 링크, 최신본/배포본 구조
- `함께문서실`의 "버전 보관함" 핵심 뼈대로 재사용 가능

4. 데스크톱 실행 경험
- `desktop/teacher-launcher`가 이미 외부 런처/프로토콜 기반 흐름을 갖고 있다
- 향후 `한글로 열기`, `한셀로 열기`도 같은 발상으로 연결 가능

5. 한글 문서 이해 보조 자산
- `hwpxchat`가 HWPX 업로드와 문서 읽기 흐름을 이미 갖고 있다
- 문서 미리보기, 요약, 텍스트 추출, "HWP는 HWPX로 바꿔 업로드" 같은 안내에 재사용 가능

정리:
- 완전히 바닥부터 새로 만드는 것이 아니라
- `문서 허브 UI + 협업 권한 + 데스크톱 연결 + Google 연동`을 하나의 서비스 경험으로 묶는 작업이다

## 4. 서비스 구조 제안

### 4-1. 서비스 가족 구조

`함께문서실`은 한 화면에서 네 가지 길을 제공한다.

1. `Google로 같이 쓰기`
- 회의록
- 업무분담표
- 명단
- 공동 체크리스트
- 학년 협업 문서

2. `한글로 열기`
- 공식 양식
- 공문 초안
- 교육청 제출 문서
- 결재 전 문서

3. `한셀로 열기`
- 학급/학년 목록표
- 업무 배정표
- 생활기록 보조표
- 행사 운영표

4. `버전 보관함`
- 최신본
- 배포본
- 이전본 비교
- 누가 언제 열고 저장했는지 기록

### 4-2. 서비스 배치 원칙

교사 입장에서 헷갈리지 않게, 문서 생성 첫 화면에서 문서 성격을 먼저 묻는다.

- `여러 사람이 지금 같이 써야 하나요?`
  - 예 -> `Google 문서/시트`
  - 아니오 -> `한글/한셀` 또는 `버전 보관함`

- `교육청 양식이나 공식 제출 문서인가요?`
  - 예 -> `한글/한셀`
  - 아니오 -> `Google 문서/시트` 또는 `간단 문서`

이 질문 두 개만으로 대부분의 교사 업무는 자연스럽게 갈린다.

## 5. 구현 원칙

### 5-1. 교사 UX 원칙

1. 문서 종류보다 `하려는 일`을 먼저 보여준다
- "회의록 같이 쓰기"
- "학년 목록 함께 정리"
- "한글 문서 최신본 관리"
- "우리 반 문서 공유"

2. 다음 행동이 항상 바로 보여야 한다
- 문서를 만들면 바로 `편집자 초대`
- 초대가 끝나면 바로 `열기`
- 저장이 끝나면 바로 `최신본 확인`

3. 용어를 개발자식으로 쓰지 않는다
- `OAuth 연결` 대신 `Google 연결`
- `동기화 에이전트` 대신 `문서 연결 도우미`
- `권한 상속` 대신 `이 문서를 함께 볼 사람`

4. 실패 시 행동이 바로 보여야 한다
- "Google 연결이 끊겼습니다. 다시 연결 버튼을 눌러 주세요."
- "한글이 열리지 않았습니다. 문서 연결 도우미가 설치되어 있는지 확인해 주세요."

5. 핵심 액션은 항상 1차 버튼 하나로 보인다
- `Google로 같이 쓰기`
- `한글로 열기`
- `한셀로 열기`
- `편집자 초대`
- `최신본 내려받기`

### 5-2. 서비스 통합 원칙

1. 신규 독립 앱으로 시작한다
- 서비스 가이드 기준에 따라 신규 서비스는 별도 앱이 기본
- 제안 앱명: `doccollab`

2. 공용 파일은 최소만 건드린다
- 홈 카드 노출, 제품 등록, 매뉴얼 노출 정도만 공용 레벨 반영

3. 기존 `version_manager`를 대체하기보다 흡수 경로를 설계한다
- 1차: 버전관리 엔진 재사용
- 2차: 함께문서실 UI에서 버전 보관함을 통합 노출

4. 교실형 맥락을 살린다
- 현재 `active classroom`과 연결
- 학급/학년 단위 문서함을 자연스럽게 선택 가능

## 6. 목표 아키텍처

### 6-1. 사용자에게 보이는 서비스

1. `함께문서실 홈`
2. `문서 만들기`
3. `문서방`
4. `버전 보관함`
5. `활동 기록`
6. `Google 문서/시트 연결 설정`
7. `문서 연결 도우미 설치 안내`

### 6-2. 내부 구성

1. 웹 허브
- 문서 목록
- 권한
- 버전
- 활동 기록
- 링크/공유

2. Google 협업 계층
- Google Docs
- Google Sheets
- 교사 개인 Google 연결
- 실시간 편집은 Google 화면을 사용하되, 진입과 관리와 검색은 eduitit가 맡는다

3. 한컴 데스크톱 연결 계층
- Windows용 `문서 연결 도우미`
- `한글로 열기`, `한셀로 열기`
- 저장/닫기 감지
- 버전 업로드
- 잠금 또는 편집 상태 표시

4. 공통 권한 계층
- 나만 보기
- 학급 공유
- 학년 공유
- 직접 초대
- 읽기만
- 수정 가능

5. 공통 기록 계층
- 누가 열었는지
- 누가 저장했는지
- 어떤 파일이 최신본인지
- 배포본이 무엇인지

### 6-3. 가장 중요한 결정

`Google은 실시간 공동편집`, `한컴은 관리형 데스크톱 편집`으로 역할을 분리한다.

이 분리가 중요한 이유:
- Google의 장점을 살린다
- 한컴의 현장 사용성을 살린다
- 교사가 어떤 도구를 써야 할지 쉽게 이해한다
- 법적/기술적 리스크를 단계적으로 줄일 수 있다

## 7. 핵심 사용자 여정

### 여정 A. 학년 회의록 같이 쓰기

1. `함께문서실` 진입
2. `회의록 같이 쓰기` 선택
3. `Google 문서` 자동 추천
4. 문서명 입력
5. 학년부 교사 체크
6. `문서 만들기`
7. 문서방에서 `Google로 열기`
8. 실시간으로 함께 수정
9. 종료 후 eduitit에서 버전/참여 기록 확인

### 여정 B. 학년 업무분담표 함께 정리

1. `함께문서실` 진입
2. `목록표 함께 정리` 선택
3. `Google 시트` 자동 추천
4. 템플릿 선택: 담당업무표 / 행사분장표 / 명단표
5. 편집자 초대
6. `시트 열기`
7. 실시간 공동 수정
8. 완료 후 `읽기 전용 공유 링크` 또는 PDF 출력

### 여정 C. 교육청 양식 문서 작성

1. `함께문서실` 진입
2. `공식 문서 작성` 선택
3. `한글로 열기` 추천
4. 문서 연결 도우미 설치 확인
5. `한글로 열기`
6. 로컬 한글에서 수정
7. 저장하면 eduitit가 최신본 반영
8. 문서방에서 `최신본`, `배포본`, `이전본 비교` 확인

### 여정 D. 한셀 목록표 최신본 관리

1. `함께문서실` 진입
2. `한셀로 관리할 목록표` 선택
3. 문서명과 그룹 지정
4. `한셀로 열기`
5. 저장 후 자동 업로드
6. 문서방에서 저장 이력과 최신본 확인
7. 필요하면 `Google 시트로 복제` 버튼으로 실시간 협업본 생성

## 8. 화면 설계

### 8-1. 홈 화면

목표:
- 교사가 5초 안에 "어디로 들어가야 하는지" 이해해야 한다

핵심 블록:
- 오늘 이어서 할 문서
- 새 문서 만들기
- Google로 같이 쓰기
- 한글/한셀로 열기
- 최근 저장된 문서
- 우리 학년 공유 문서

### 8-2. 문서 만들기 화면

입력 순서:
1. 무엇을 하시나요?
2. 누구와 같이 쓰나요?
3. 어떤 방식으로 열까요?
4. 문서명/그룹/학급
5. 바로 열기

문서 유형 카드:
- 회의록 같이 쓰기
- 목록표 함께 정리
- 공식 한글 문서 작성
- 한셀 목록 관리
- 자료 보관만 하기

### 8-3. 문서방 화면

핵심 레이아웃:
- 왼쪽: 문서 정보, 참여 교사, 공유 범위
- 가운데: 지금 해야 할 일, 열기 버튼, 최근 활동
- 오른쪽: 최신본/배포본/이전본, 댓글/메모, 체크리스트

문서방에서 절대 숨기면 안 되는 것:
- 지금 어떤 방식으로 열어야 하는지
- 누가 편집 중인지
- 최신 저장 시각
- 다음 행동 버튼

### 8-4. 설정 화면

필수 설정만 둔다.

- Google 연결
- 문서 연결 도우미 설치 확인
- 기본 저장 폴더
- 기본 공유 범위
- 알림 받기 여부

## 9. UI 카피 제안

### 홈 카드 문구

- `Google로 같이 쓰기`
  - "회의록, 명단, 분담표를 여러 선생님이 동시에 수정하세요."

- `한글로 열기`
  - "공식 문서는 내 컴퓨터의 한글로 바로 열고 최신본만 자동 관리하세요."

- `한셀로 열기`
  - "목록표는 익숙한 한셀로 정리하고, 저장할 때마다 최신본을 남기세요."

- `버전 보관함`
  - "최종최최종 파일명을 그만 쓰고, 최신본과 배포본을 분리해 보관하세요."

### 상태 문구

- `편집 중`
- `저장 중`
- `저장 완료`
- `배포본 확정`
- `다음 할 일: 편집자 초대`
- `다음 할 일: Google로 열기`
- `다음 할 일: 한글로 열기`

### 오류 문구

- `Google 연결이 끊겼습니다. 다시 연결해 주세요.`
- `문서 연결 도우미가 아직 설치되지 않았습니다. 1분 설치 후 다시 열어 주세요.`
- `저장 내용을 아직 받지 못했습니다. 네트워크를 확인한 뒤 다시 저장해 주세요.`

## 13. Google Docs/Sheets를 함께 넣는 것에 대한 판단

결론:
- `넣는 것이 맞다`
- 다만 `함께문서실` 안에서 역할을 명확히 나눠야 한다

넣어야 하는 이유:
1. 교사 공동 목록 관리의 즉시 체감 가치가 크다
2. 한컴보다 실시간 공동편집 경험이 안정적이다
3. 명단, 일정 합치기, 분담표, 점검표는 Google Sheets가 현장 효율이 높다
4. 한컴 연동이 늦어져도 서비스 가치를 먼저 출시할 수 있다

주의할 점:
1. Google과 한컴을 같은 버튼으로 섞지 않는다
2. 문서 생성 단계에서 추천 경로를 분명히 나눈다
3. 교사가 "왜 이 도구로 열리는지" 이해할 수 있어야 한다

추천 정책:
- 회의록/협업 목록/공동 초안 -> Google Docs/Sheets 기본 추천
- 공식 문서/제출 양식/보안 문서 -> 한글/한셀 기본 추천

## 19. 최종 권장안

가장 좋은 실행 순서는 아래와 같다.

1. `함께문서실` 독립 서비스로 시작한다

3. 문서방/권한/버전/활동 기록을 함께 만든다
4. 그 다음 한글/한셀 `문서 연결 도우미`를 붙인다
5. 마지막으로 한컴 저장 자동반영과 배포본 확정까지 통합한다

이 순서가 좋은 이유:
- 교사 체감 가치를 빨리 만든다
- 실시간 협업 수요를 먼저 해결한다
- 한컴 연동의 법적/기술적 리스크를 분리한다
- 최종적으로는 `eduitit가 문서 허브`가 된다

## 20. 이 계획의 결과물 정의

이 계획이 완료되면 `eduitit`는 단순한 서비스 모음이 아니라,

- `Google로 같이 쓰는 문서`
- `한글/한셀로 다루는 공식 문서`
- `최신본과 배포본을 관리하는 문서함`

을 하나의 교사 경험으로 묶는 `문서 운영 허브`로 확장된다.

교사 입장에서 기억해야 할 문장은 하나면 된다.

`문서는 eduitit에서 만들고, 가장 편한 방식으로 열고, 최신본은 eduitit가 책임진다.`
