# CHECKLIST: Sheetbook / Calendar / Message Final Architecture

작성일: 2026-03-08
대상 서비스: `sheetbook`, `classcalendar`
목표 수준: 설명 없이도 사용하는 엔터프라이즈급 교사용 서비스

## 0. 최종 원칙

- [ ] 달력은 하나만 둔다.
- [ ] `sheetbook` 안 달력은 별도 서비스처럼 보이지 않는다.
- [ ] `iframe`, 이중 스크롤, 중복 상단바를 제거한다.
- [ ] 설명문보다 구조와 배치만으로 사용 흐름이 읽히게 만든다.
- [ ] `sheetbook`, `calendar`, `message`는 같은 데이터 원본과 같은 상세 흐름을 사용한다.

## 1. 정보구조

- [ ] `sheetbook`의 핵심 탭을 `표`, `달력`, `기록` 기준으로 정리한다.
- [ ] `달력` 탭 상단에는 문맥 배지와 핵심 액션만 남긴다.
- [ ] `표` 탭의 상단 안내 박스는 축소하거나 제거한다.
- [ ] 설정성 기능은 작업 화면 전면에서 빼고 별도 설정 영역으로 보낸다.

## 2. 달력 구조

- [ ] `classcalendar`를 `페이지 shell`과 `달력 body`로 분리한다.
- [ ] `달력 body`는 상단 네비게이션 없이 독립 렌더 가능해야 한다.
- [ ] `sheetbook`은 `달력 body`를 직접 렌더한다.
- [ ] `sheetbook` 달력 탭에서 `iframe`과 `postMessage`를 제거한다.
- [ ] 이벤트 상세, 첨부, 수정, 삭제, message capture는 같은 모달/상태를 사용한다.

## 3. 표 UX

- [ ] 새 표는 기본 30줄로 시작한다.
- [ ] 마지막 5줄 근처 입력 시 20줄 자동 추가가 유지된다.
- [ ] 항목 이름 변경은 헤더 인라인 편집으로 유지한다.
- [ ] 줄 삭제는 행 번호 셀 근처에서 한다.
- [ ] 항목 삭제는 헤더 셀 근처에서 한다.
- [ ] 데스크톱 삭제 버튼은 hover/focus에서만 노출한다.
- [ ] 모바일은 하단 액션 패널로 간단 편집만 허용한다.

## 4. 메시지 흐름

- [ ] 메시지 붙여넣기는 달력의 일부 기능으로 유지한다.
- [ ] 흐름은 `붙여넣기 -> 검토 -> 저장`으로 유지한다.
- [ ] 저장 후 같은 이벤트 상세로 바로 이어진다.
- [ ] 이벤트 상세에서 원본 수첩/탭을 다시 열 수 있다.

## 5. 문구와 버튼

- [ ] `학급 달력`, `교무수첩 달력`, `캘린더` 잔여 문구를 핵심 화면에서 제거한다.
- [ ] 버튼은 `바로 실행`, `다음 화면`, `검토 후 저장`으로 성격이 읽혀야 한다.
- [ ] 핵심 화면에 기술 용어가 남지 않게 한다.
- [ ] 실패 문구는 원인과 다음 행동을 함께 보여준다.

## 6. 모바일

- [ ] 모바일에서도 같은 구조를 유지하되 액션 밀도만 줄인다.
- [ ] 허용: 칸 1개 수정, 줄 추가/삭제, 항목 이름 변경, 간단한 항목 추가/삭제, 새 일정, 메시지 붙여넣기
- [ ] 비허용: 대량 붙여넣기, 다중 범위 선택, 복잡한 일괄 작업
- [ ] 비허용 액션은 비활성화와 이유 안내를 함께 제공한다.

## 7. 접근성 / 운영 가드

- [ ] 키보드만으로 표/달력 핵심 흐름 1회 수행 가능
- [ ] 모달 ESC 닫기와 포커스 복귀 보장
- [ ] 주요 생성/연결 액션 로그와 메트릭 유지
- [ ] 권한 경계 테스트 유지
- [ ] 원본 서비스 이동 경로와 audit trace 유지

## 8. 와이어프레임 기준

### 표 탭
- [ ] 수첩 제목
- [ ] `표 / 달력 / 기록` 탭
- [ ] 찾기 1줄
- [ ] 표 본문
- [ ] 행/열 컨텍스트 액션

### 달력 탭
- [ ] 문맥 배지: 현재 수첩 / 연결된 일정 원본
- [ ] 핵심 액션 3개: `표 반영`, `새 일정`, `메시지 붙여넣기`
- [ ] 달력 본문
- [ ] 이벤트 상세 모달

### 메시지 모달
- [ ] 입력
- [ ] 자동 읽기
- [ ] 검토
- [ ] 저장

## 9. 스모크 기준

- [ ] Desktop: `표 작성 -> 달력 반영 -> 상세 -> 원본 복귀`
- [ ] Desktop: `메시지 붙여넣기 -> 일정 생성 -> 첨부 확인`
- [ ] Tablet: 달력 본문 / 새 일정 / 메시지 붙여넣기 정상
- [ ] Mobile: 새 일정 / 메시지 붙여넣기 / 간단 편집 정상
- [ ] 중복 상단바 0
- [ ] 이중 스크롤 0
- [ ] 콘솔 `ReferenceError` 0
- [ ] 치명 경고 0

## 10. 구현 순서

- [ ] 1. `classcalendar` shell/body 분리
- [ ] 2. `sheetbook` 달력 탭에서 iframe 제거
- [ ] 3. 공용 모달/상태 연결
- [ ] 4. 표 화면 상단 구조 정리
- [ ] 5. 잔여 용어 정리
- [ ] 6. 스모크/QA 갱신
