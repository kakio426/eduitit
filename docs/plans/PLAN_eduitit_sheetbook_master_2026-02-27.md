# Eduitit 교무수첩(시트북) 마스터 플랜

- 작성일: 2026-02-27
- 작성 목적: Eduitit의 차세대 메인 경험을 `서비스 카탈로그` 중심에서 `교사 작업공간(교무수첩)` 중심으로 전환
- 핵심 전략: 추천 1순위(로그인 후 대시보드 전환) + 추천 3순위(학급 캘린더 승격) 결합

---

## 1. 문제 정의

현재 교사들은 구글시트 1개 파일 안에 여러 탭을 만들어 아래 업무를 한 번에 처리한다.

- 연간/월간 일정
- 학생명부
- 외부강사 수업 관리
- 생태체험/부모 도우미
- 사서/보건 수업 기록
- 메모와 임시 기록 보관

문제는 Eduitit의 강력한 서비스들이 있어도, 시작점이 분산되어 있어 “한 번에 처리” 체감이 약하다는 점이다.

---

## 2. 성공 기준

### 2.1 제품 성공 기준

- 구글시트보다 복잡하지 않다.
- 교사가 계획을 먼저 세우지 않아도, 복붙 중심으로 바로 쓸 수 있다.
- 기존 Eduitit 서비스(간편수합, 동의서, 서명, 캘린더 등)와 자연스럽게 연결된다.

### 2.2 운영 지표(KPI)

- 첫 로그인 후 10분 내 교무수첩 생성률 60%+
- 교무수첩 생성 사용자 중 7일 내 재방문률 45%+
- 교무수첩 내 액션(안내문/동의서/수합/달력등록) 실행률 30%+
- 교무수첩 사용 교사의 “업무 시간 단축 체감” 설문 긍정 70%+

---

## 3. 제품 원칙

- 원칙 1: 원본은 시트, 기능은 액션
- 원칙 2: 입력은 자유롭게, 연결은 원클릭으로
- 원칙 3: 기본 화면은 단순하게, 고급 기능은 늦게
- 원칙 4: 강요되는 스키마 금지, 필요한 순간의 제안만 허용
- 원칙 5: 내보내기/가져오기는 항상 열어둔다

금지 범위(MVP):

- 수식/피벗/복잡한 조건부 포맷팅
- 초기 온보딩에서 긴 설정 마법사
- 홈 화면의 과도한 도구 나열

---

## 4. 최종 UX 방향

## 4.1 로그인 전 홈

- 현재 구조 유지(서비스 탐색 중심)
- 상단에 `교무수첩` 홍보 블록 1개 추가
- CTA: `로그인하고 시작` / `미리보기`

## 4.2 로그인 후 홈(신규 대시보드)

홈의 기본 목적을 “서비스 찾기”에서 “오늘 업무 시작”으로 전환한다.

우선 노출 섹션:

- 오늘
- 내 교무수첩
- 학급 운영 수첩(달력 포함) 진입 카드
- 빠른 실행(핵심 4개)

## 4.3 교무수첩 내부

기본 탭 4개:

- 달력
- 일정
- 학생명부
- 메모

추가 탭은 추천형:

- 외부강사
- 체험학습/도우미
- 사서/보건 수업
- 예산/정산

핵심은 “정형화 강요”가 아니라 “탭을 자유롭게 만들고 필요 시 기능 연결”이다.

## 5. 정보 구조(IA)

- Workspace(Home, 로그인 후)
- Sheetbook List
- Sheetbook Detail
- Tab(Grid)
- Action Layer
- Output(안내문/동의서/수합/체크/캘린더)

권장 네비게이션:

- 상단 고정 버튼: `+ 새 교무수첩`, `가져오기`
- Ctrl+K 검색 결과에 `교무수첩/탭/최근 행` 포함

## 6. 핵심 기능 범위

## 6.1 시트북 공통

- 탭 생성/복제/이름변경/정렬
- 행/열 추가, 복붙, 기본 필터/정렬
- 전체 검색(시트북 단위)
- 저장된 보기(필터 프리셋)
- 자동 저장

## 6.2 학급 캘린더 승격

- 교무수첩 내부 `달력 탭` 기본 제공
- 일정 탭에서 날짜 감지 후 원클릭 캘린더 반영
- 기존 classcalendar와 양방향 최소 동기화(충돌 시 최근 수정 우선 정책)

## 6.3 액션 레이어(선택 기반)

시트에서 범위 선택 시만 노출:

- 안내문 만들기
- 동의서/설문 만들기
- 간편수합 만들기
- 출석/체크 만들기
- 달력 등록

## 6.4 가져오기/내보내기

- CSV/엑셀 업로드
- 구글시트 복붙 파서
- CSV/엑셀/PDF 내보내기

## 7. 기술 구현 방향(초안)

## 7.1 신규 앱 제안

- 앱명 제안: `sheetbook` (또는 `workspace`)

핵심 모델:

- `Sheetbook` (소유자, 제목, 학년도, 공개범위, 고정 여부)
- `SheetTab` (시트북 FK, 이름, 타입[grid/calendar], 정렬)
- `SheetColumn` (탭 FK, 타입, 라벨, 정렬, 옵션 JSON)
- `SheetRow` (탭 FK, 정렬, 생성/수정 메타)
- `SheetCell` (row+column FK, value_text/value_number/value_json)
- `SavedView` (탭 FK, 필터/정렬 JSON)
- `ActionInvocation` (어떤 범위를 어떤 서비스로 보냈는지 로그)

## 7.2 기존 서비스 연동 포인트

- `classcalendar`: 달력 탭 동기화
- `collect`: 선택 범위 기반 수합 생성
- `consent`: 선택 범위 기반 동의서 생성
- `signatures`: 선택 범위 기반 서명 요청 생성
- `handoff`: 공유 명단 그룹 가져오기/내보내기

## 7.3 API/성능 원칙

- 큰 시트는 페이지네이션/가상 스크롤
- 셀 단위 저장 대신 배치 저장(300~500ms 디바운스)
- 검색은 탭/시트북 단위 인덱싱

## 8. 단계별 로드맵

## Phase 0. 기획 고정 (1주)

- 기능 원칙 확정(복잡도 상한 포함)
- IA/용어 통일(교무수첩/탭/액션)
- 프로토타입 검토(교사 5명 내외)
- 완료 기준: MVP 범위와 비범위 합의

## Phase 1. 기반 구축 (2~3주)

- `sheetbook` 데이터 모델/마이그레이션
- 기본 권한/공유 정책
- 시트 그리드 기본 조작(행/열/복붙)
- 완료 기준: 데이터가 안전하게 쌓이고 다시 열린다

## Phase 2. 로그인 후 홈 전환 (2주)

- 로그인 후 홈을 작업공간형으로 분기
- `오늘`, `내 교무수첩`, `학급 운영 수첩` 카드 배치
- 빠른 실행 4개 배치
- 완료 기준: 로그인 사용자의 기본 진입이 교무수첩 중심

## Phase 3. 학급 캘린더 승격 (2~3주)

- 교무수첩 내부 달력 탭 탑재
- 일정 탭-달력 탭 연동
- 기존 classcalendar URL 호환 라우팅
- 완료 기준: 캘린더가 “별도 서비스”가 아니라 “수첩 내부 탭”으로 동작

## Phase 4. 액션 레이어 연결 (3~4주)

- 선택 범위 감지 UI
- 안내문/동의서/수합/체크/캘린더 등록 연결
- 실행 결과를 교무수첩에 링크 형태로 되돌려 표시
- 완료 기준: 교무수첩에서 1클릭으로 기존 핵심 서비스 호출 가능

## Phase 5. 가져오기/검색/출력 강화 (2~3주)

- CSV/엑셀/복붙 가져오기 안정화
- 시트북 통합 검색
- 저장된 보기, 내보내기
- 완료 기준: 구글시트 대체 진입 장벽 최소화

## Phase 6. 파일럿/정식 론칭 (2주)

- 교사 파일럿(학교/학년/학급 다변화)
- 주요 퍼널/오류/이탈 지점 보완
- 가이드 문서/튜토리얼 배포
- 완료 기준: 베타 플래그 해제 가능 상태

- 파일럿 교사 만족도 기준 충족

## 13. 의사결정 로그(초기)

- 결정: 정형화된 업무 모듈 우선 전략을 배제하고, 시트북 자유입력 전략 채택
- 결정: 로그인 후 홈의 기본 진입점을 교무수첩으로 전환
- 결정: 학급 캘린더는 별도 서비스가 아니라 교무수첩 내부 기본 탭으로 승격
- 보류: 실시간 공동편집 범위(초기에는 단일 사용자 중심으로 시작)

---

## 14. 결론

이 프로젝트의 핵심은 “기능 추가”가 아니라 “진입 경험의 전환”이다.

- Before: 서비스를 찾아 들어가는 구조
- After: 교무수첩에서 일을 하다가 필요할 때 서비스를 붙이는 구조

즉, Eduitit을 교사용 앱 모음에서 교사용 업무 운영체계로 바꾸는 가장 큰 전환 프로젝트로 정의한다.
