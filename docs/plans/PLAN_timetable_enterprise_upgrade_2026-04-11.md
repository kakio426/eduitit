# 우리학교 시간표 엔터프라이즈 전환 계획

## 현재 문제 목록
- 첫 화면이 관리자 중심이라 담임과 업무 담당 교사가 어디로 들어가야 하는지 바로 알기 어렵다.
- 사용자 화면에 구현 기술명과 설명성 문구가 남아 있어 업무 흐름보다 도구 설명이 먼저 보인다.
- 관리자 편집 화면은 기능은 충분하지만 `확정 가능 상태`, `검토 필요 반`, `최근 활동`이 한눈에 보이지 않는다.
- 반별 입력 링크 운영은 가능하지만 링크 상태, 마지막 접근, 검토 흐름이 운영 화면에서 더 선명해야 한다.
- 확정은 충돌만 막고 있어 운영 검토가 끝났는지까지는 한 번에 판단하기 어렵다.
- 저장/복원/발급/검토 같은 핵심 작업의 감사 흔적이 부족하다.

## 남길 화면 / 내릴 화면 / 통합할 화면
- 남길 화면
  - `/timetable/`
  - `/timetable/workspaces/<id>/`
  - `/timetable/workspaces/<id>/meeting/`
  - `/timetable/edit/<token>/`
  - `/timetable/share/<token>/`
  - `/timetable/share/portal/<token>/`
- 내릴 화면
  - 교사 관리, 특별실 정책, 레거시 가져오기는 메인 편집 화면의 전면 영역에서 내리고 보조 설정으로 유지
- 통합할 화면
  - 메인에서 관리자 시작, 담임 링크 진입, 확정본 링크 진입을 한 허브에서 처리
  - 관리 화면에서 충돌, 검토 현황, 확정 가능 상태, 최근 활동을 한 맥락으로 묶기

## 텍스트 와이어프레임
```text

담임 / 업무 담당
[반 입력 링크 열기]
[확정본 링크 열기]

진행 현황
- 검토 필요 2개
- 입력 중 3개
- 확정본 1개
```

```text

[확정] [스냅샷]

좌측
- 현재 선택한 칸
- 빠른 배정
- 주요 충돌
- 반별 입력 현황

중앙
- 학년 전체 편집기

- 시수
- 공통 행사
- 확정 가능 상태
- 최근 활동

```

```text
오른쪽

- 반영 전 확인
- 불가 사유
- 반영 결과
```

## 구현 체크리스트
- 메인 허브를 역할 중심으로 재배치하고 구현 기술명 노출 제거
- 반 입력 링크 / 확정본 링크 직접 진입 폼 추가
- 반별 입력 상태 요약, 검토 필요 수, 최근 검토 목록 계산 추가
- `TimetableAuditLog` 추가와 핵심 작업 로그 기록
- `publish_readiness`, `progress_summary`, `recent_activity` 서버 계약 추가
- publish 시 `검토 미완료 반` 차단과 class status `published` 반영
- 관리자 편집 화면에 상태 바, 확정 준비 패널, 최근 활동 패널 추가
- 반별 입력 현황에 검색, 링크 상태, 마지막 접근 표시 추가
- 회의 화면에 반영 전 요약과 불가 사유 고정 패널 강화
- 제품 카피와 매뉴얼을 사용자 업무 표현 중심으로 정리
