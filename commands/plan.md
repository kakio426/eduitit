# 작업 계획 수립

새로운 기능이나 복잡한 작업을 시작하기 전에 계획을 세웁니다.

## 계획 수립 단계
1. **요구사항 분석**: 무엇을 구현해야 하는가?
2. **현재 코드 파악**: 관련 코드베이스 탐색
3. **접근 방식 결정**: 가능한 구현 방법들 비교
4. **작업 분해**: 구체적인 단계로 나누기
5. **위험 요소 식별**: 예상되는 문제점

## 출력 형식
```markdown
## 요구사항 요약
- ...

## 구현 계획
1. 단계 1: ...
2. 단계 2: ...

## 수정할 파일
- 파일1: 변경 내용
- 파일2: 변경 내용

## 예상 위험 요소
- ...
```

계획이 승인되면 auto-accept 모드로 전환하여 구현합니다.
